# Gemini Independent Review (Second Opinion)

This review was generated by running the Gemini CLI (`gemini`, v0.51.0) in
headless mode (`gemini --skip-trust -p ...`), pointed at the full contents of
this repo's blog core (`js/`, `styles/`, `index.html`, `post.html`) and both
mini apps (`app2048/`, `apppixelart/`). It is an independent, fresh pass
meant to complement — not repeat — the prior Claude-based reviews already on
record at `app2048/review.md` and `apppixelart/review.md`, which covered live
browser functional testing (keyboard/touch input, win/game-over overlays,
localStorage persistence, responsive layout, contrast fixes, etc.). Gemini
was explicitly briefed on those prior findings and asked to focus on what
that browser-driven pass would not have caught: static-code security review,
edge-case correctness bugs, deeper accessibility gaps, and code-quality/
duplication issues.

**Note on model used:** the session's assigned API key had its daily quota
exhausted on `gemini-3.5-flash` (the CLI's default routing target for
`gemini-2.5-flash`/`gemini-2.0-flash`/etc. requests under this account), and
`gemini-2.5-pro` was unavailable (quota limit 0) on this free-tier key. The
review below was produced with `gemini-3.1-flash-lite`, the only model this
key could reach that was not subject to the same routing/quota block. Findings
are Gemini's own output, reproduced faithfully and unedited below.

## Independent Code Review - my-blog

This review focuses on security, correctness, accessibility, and code quality, complementing the prior functional/contrast-focused pass.

### Security

| File path | Line number(s) / Excerpt | Severity | Description |
| :--- | :--- | :--- | :--- |
| `js/markdown.js` | `renderInline` function, regex for links/images | Critical | **XSS Vulnerability**. While `escapeHtml()` is applied to the raw text, it is not applied to the `url` captured by regex in markdown links (`[]()`) or images (`![]()`). A malicious post can use `[x](javascript:alert(1))` or `![x](javascript:alert(1))` to execute arbitrary JavaScript. |

### Correctness

| File path | Line number(s) / Excerpt | Severity | Description |
| :--- | :--- | :--- | :--- |
| `post.html` | `<html>` tag | Low | Missing `lang="en"` attribute, unlike `index.html`. This impacts screen readers and browser translation tools. |

### Accessibility

| File path | Line number(s) / Excerpt | Severity | Description |
| :--- | :--- | :--- | :--- |
| `app2048/app.js` | `showOverlay()` / `hideOverlay()` | High | **Focus Trapping**. The win/game-over overlay does not trap focus. A keyboard user (Tab) can navigate to elements behind the modal while it is open. Focus should be moved to the primary button upon opening and constrained to the modal. |
| `apppixelart/editor.js` | Canvas event listeners | Medium | **Keyboard Operability**. The canvas drawing surface is entirely reliant on pointer events. There is no fallback for keyboard users to paint pixels (e.g., using arrow keys to move a cursor and a key to toggle the current color/eraser). |

### Code Quality / Spec Compliance

| File path | Line number(s) / Excerpt | Severity | Description |
| :--- | :--- | :--- | :--- |
| `js/list-page.js` & `js/post-page.js` | `formatDate` function | Nitpick | Duplicate implementation. Should be moved to a shared `js/utils.js` file. |
| `styles/` & subfolders | CSS Token blocks | Low | Significant duplication of CSS custom properties (tokens) in every app's `styles.css` file. This makes managing design system changes (e.g., adding a color) error-prone across the project. |
| `index.html` & `post.html` | Theme FOUC script | Nitpick | Duplicate inline script to prevent FOUC. This could be moved to a tiny shared script file or handled in `theme.js`. |

---

### Summary
The project is structurally sound, clean, and fulfills the "self-contained" requirement well. However, the manual markdown parser is vulnerable to XSS via `javascript:` URIs in link/image markdown. Additionally, while functional, the mini-apps lack the accessibility rigor (focus trapping and alternative inputs) required for a fully accessible portfolio. There is also unnecessary code and configuration duplication that will hinder maintainability as the project grows.

### Prioritized Top-3 Fixes
1. **Fix XSS in `js/markdown.js`**: Implement URL sanitization (e.g., ensuring `url` starts with `http`, `https`, or `/`) within the link and image regex replacements, or add an `escapeUrl()` function to complement `escapeHtml()`.
2. **Implement Focus Trapping in `app2048`**: When an overlay opens, save the previously focused element, move focus to the primary button inside the overlay, and add a listener to keep focus within the overlay until it is closed.
3. **Consolidate Duplicated Assets**: Move the duplicated `formatDate` utility and the CSS token definitions into shared files (e.g., `js/utils.js` and `styles/tokens.css` imported where needed) to ensure a single source of truth and reduce technical debt.

## 검토 및 조치 내역 (Claude, 이 리뷰를 읽고 직접 확인·수정)

각 지적을 실제 코드와 대조해 타당성을 확인한 뒤, 타당한 것만 수정했다.

| 지적 | 판단 | 조치 |
| :--- | :--- | :--- |
| `js/markdown.js` XSS (Critical) | **타당함, 확인됨.** `[x](javascript:alert(1))`을 실제로 렌더링해보니 `<a href="javascript:alert(1)">`가 그대로 생성됨을 확인. `escapeHtml()`은 URL의 특수문자가 아닌 스킴을 막지 못한다. | **수정함.** `sanitizeUrl()` 추가 — `https?:`/`mailto:`/`/`/`#`/`.`로 시작하는 URL만 통과, 나머지는 `#`로 무력화. `javascript:` 페이로드 두 건 모두 `#`로 바뀌는 것을 브라우저에서 재확인. |
| `post.html` `lang` 속성 누락 (Low) | **타당하지 않음.** 실제 `post.html:2`는 `<html lang="en">`으로 이미 존재함 — Gemini가 잘못 판단(할루시네이션으로 추정). | 조치 없음. |
| `app2048` 포커스 트래핑 부재 (High) | **타당함.** 오버레이가 열려도 포커스 이동/트랩이 전혀 없어 Tab으로 뒤 배경 요소에 접근 가능했음. | **수정함.** 오버레이를 열기 전 포커스 요소를 저장하고, 열릴 때 주 버튼에 포커스, Tab/Shift+Tab을 오버레이 내부 버튼끼리만 순환하도록 트랩 추가, 닫을 때 리스너 제거 + 원래 포커스로 복원. `.focus()`가 실제로 동작하는지 브라우저에서 직접 확인(계산된 `display: flex`, `focusedAfterShow: true`). |
| `apppixelart` 캔버스 키보드 조작 불가 (Medium) | **타당한 관찰이지만, 지금 고치지 않음.** 실제로 캔버스가 포인터 이벤트로만 동작하는 것은 맞다. 다만 키보드로 칸을 그릴 수 있게 하려면 커서 표시 오버레이, 방향키 이동, 칠하기/지우기 키 바인딩 등 새 UI/상호작용을 새로 설계해야 해서 "버그 수정"이 아니라 "신규 기능"에 가깝다고 판단해 이번 조치에서는 보류했다. 사용자가 원하면 별도 작업으로 진행 가능. | 조치 보류. |
| `formatDate` 중복 (Nitpick) | **타당함.** `js/list-page.js`와 `js/post-page.js`에 동일한 함수가 그대로 중복돼 있었음. | **수정함.** `js/format.js`로 추출, 두 파일 모두 import로 교체. 두 페이지 모두 브라우저에서 정상 동작 재확인(글 목록 4개, 상세 페이지 날짜 표시). |
| CSS 토큰 중복 (Low) | **타당하지 않음(의도된 설계).** 각 앱을 `/app{이름}/` 안에 자체 완결시키고 블로그의 `styles/`를 직접 import하지 않는 것은 CLAUDE.md의 명시적 규칙("모든 웹앱은 /app{앱이름}/폴더 안에 자체 완결한다")이자 `app2048`/`apppixelart`의 BUILD.md 지침에서 의도적으로 내린 결정이다(파일 구조 결합 방지). 토큰을 공유 파일로 합치면 앱의 자체 완결성이 깨진다. | 조치 없음. |
| 테마 FOUC 스크립트 중복 (Nitpick) | **타당하지 않음(의도된 설계).** `js/theme.js` 파일 자체의 주석에 이미 "이 인라인 스크립트는 FOUC 방지를 위해 `<head>`에서 동기적으로 실행돼야 한다"고 명시돼 있다. 외부 파일로 뽑으면 네트워크 요청이 추가로 필요해 오히려 FOUC 방지 목적에 역행할 수 있어, 6줄짜리 중복을 감수하는 편이 낫다고 판단. | 조치 없음. |

**결론**: 7건 중 3건 수정(XSS, 포커스 트랩, formatDate 중복), 1건 보류(피드백은 남기되 신규 기능이라 스코프 밖), 3건은 타당하지 않음(오탐 1건, 의도된 설계 2건)으로 판단해 조치하지 않음.
