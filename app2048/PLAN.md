# 지침 파일 — Plan 단계 (app2048)

너는 my-blog 프로젝트의 새 미니 웹앱 "2048 게임"의 **Plan 서브에이전트**다. 이 파일이 네게 주어진 전용 지침 파일이며, 여기 명시된 범위만 수정한다.

## 배경
- 프로젝트 루트: `C:\Users\tagi7\Desktop\claude_exam\my-blog`
- 이 프로젝트의 `CLAUDE.md`를 반드시 먼저 읽어라. Plan→Build→Review→Embed→Commit 워크플로, 서브에이전트 규칙, 웹앱 규칙, 앱 디렉토리 구조, Build 서브에이전트 금지 파일 목록이 정의돼 있다.
- 사용자 요청 원문: "2048게임을 만들어줘. 방향키로 숫자 타일을 밀어서 합치는 퍼즐 게임이야. 점수판도 넣어줘."

## 네가 할 일
1. `C:\Users\tagi7\Desktop\claude_exam\my-blog\app2048\spec.md` 파일을 작성한다. 이것이 유일한 산출물이다 — **코드는 한 줄도 작성하지 않는다.**
2. spec.md에는 다음이 들어가야 한다:
   - 앱 개요 (2048 게임 규칙 요약: 4x4 그리드, 방향키로 타일 이동/병합, 같은 숫자 타일이 만나면 합쳐지며 점수 증가, 매 턴마다 빈 칸에 새 타일(2 또는 4) 랜덤 생성, 2048 타일 생성 시 승리, 더 이상 움직일 수 없으면 게임오버).
   - 화면 구성: 이 게임은 단일 화면(보드 + 점수판 + 새 게임 버튼 + 승리/게임오버 오버레이)으로 설계하고, 화면이 3개 미만이므로 Build 서브에이전트를 화면별로 나눌 필요가 없음을 명시. (화면이 실제로 3개 이상이라고 판단되면 그 근거와 함께 화면별 분리안을 제시해도 됨.)
   - 파일 구조: `/app2048/` 폴더 안에 자체 완결. 예: `index.html`, `styles.css`, `game.js` (또는 합리적으로 판단한 구조). 외부 라이브러리는 쓰지 않는 방향을 기본으로 하되(2048 게임 로직은 순수 JS로 충분), CDN을 꼭 써야 할 이유가 있다면 spec.md에 근거를 적을 것.
   - 게임 로직 설계 요약: 4x4 배열 상태 표현, 방향별(상/하/좌/우) 이동+병합 알고리즘 개요, 점수 계산 규칙(합쳐진 타일 값만큼 점수 가산), localStorage를 이용한 최고 점수(best score) 저장 여부 제안.
   - 조작: 키보드 방향키(필수) + 모바일 터치 스와이프(CLAUDE.md 웹앱 규칙 "모바일에서도 사용할 수 있어야 한다"를 충족하기 위해 필수)를 모두 지원하도록 설계.
   - my-blog의 기존 디자인 톤(에디토리얼, 화이트/다크 캔버스, near-black 프라이머리 버튼, hover 없음, 라운드 코너 위계 등— `CLAUDE.md`가 참조하는 `DESIGN-airtable.md`와 `styles/tokens.css`를 참고)과 시각적으로 크게 어긋나지 않게 하되, 게임 보드/타일 색상은 2048 특유의 색상 팔레트를 써도 된다는 점을 spec에 명시(게임 장르 특성상 완전히 동일한 톤을 강제하진 않음). 다크모드 대응 여부도 명시(가능하면 지원 권장).
3. Build 서브에이전트에게 넘길 `BUILD.md` 지침 파일의 초안도 spec.md 안에 포함하거나 별도로 언급한다 (실제 BUILD.md 파일 생성은 사용자 승인 후 별도 단계에서 하므로 지금은 만들지 않는다 — spec.md 안에 "Build 단계에서 이런 지침을 줄 예정"이라고만 적어도 됨).

## 범위 제한 (중요)
- **오직 `C:\Users\tagi7\Desktop\claude_exam\my-blog\app2048\spec.md` 파일 하나만 생성한다.**
- `index.html`, `js/`, `styles/`, `posts/`, `scripts/`, `CLAUDE.md`, `DESIGN-airtable.md`, `README.md`, `package.json`, `.gitignore`, `.claude/` 등 블로그의 다른 어떤 파일도 읽는 것 외에는 건드리지 않는다(읽기는 참고용으로 허용, 쓰기/수정 금지).
- 코드(HTML/CSS/JS)를 작성하지 않는다. 이번 단계는 계획뿐이다.
